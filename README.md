# 🚨 SOS Gateway Telegram v3

ESP32-C3 및 OLED를 기반으로 제작된 **스마트 홈 긴급 호출 게이트웨이 펌웨어**입니다. 
집 안 어디서든 433MHz RF 통신 기반의 아날로그 SOS 버튼을 누르면, ESP32가 물리적 신호를 수신한 뒤 가족들의 텔레그램 단톡방으로 즉시 스마트폰 푸시 알림을 전송하는 사물인터넷(IoT) 장치 코어 시스템입니다.

---

## ✨ 핵심 기능 (Features)

1. **텔레그램 API 기반 즉각 알림 전송** 📡
   - 지정된 텔레그램 톡방으로 즉시 위급 상황 메시지와 어떤 버튼(식별 번호)이 눌렸는지 전송.
   - 마크다운 문법 오류에 대비한 안전한 Plain Text 방식 전송 시스템.

2. **직관적인 OLED 디스플레이 (72x40) UI 디자인** 📺
   - 와이파이 상태, 버튼 페어링 모드, 알림 발송 중(종이비행기 아이콘), 에러 메시지(X 아이콘) 등을 직접 컴파일된 기하학 및 XBM 아이콘으로 출력해 현재 상태를 사용자에게 명확하게 인지시킵니다.

3. **스마트 다크 모드 설정 포털 (WiFiManager 연동)** 📶
   - 소스코드 수정 없이도 언제든 기기의 관리자 와이파이(`SOS_Gateway_Setup`)에 접속하여 새로운 가정용 인터넷 공유기를 등록하고, 텔레그램 봇 토큰과 채팅방 ID를 변경할 수 있습니다. 

4. **직관형 물리 본체 버튼 및 LED 피드백** 🔘
   - 물리 버튼 하나로 **[짧게 누르기]**: 새로운 SOS RF 버튼 페어링 (최대 20개). 실수 방지를 위한 자동 롤백(Rollback) 백업/복구 지원.
   - **[3초 길게 누르기]**: 설정 강제 초기화 및 Wi-Fi 재설정 모드 진입.
   - 동작에 맞는 Status LED 틱(Tick) 점멸 피드백 제공.

5. **강력한 불법 복제 방지 (Anti-Cloning License)** 🔒
   - 펌웨어를 무단으로 복제하는 것을 막기 위해, ESP32의 고유 기기 MAC 주소와 서버의 `SECRET_KEY`를 결합해 인증합니다. 라이선스 키가 맞지 않으면 시스템 부팅이 멈추고 화면에 **[라이선스 오류 자물쇠]** 가 표시되며 작동을 거부합니다.

6. **텔레그램 챗봇을 통한 무결성 원격 업데이트 (GitHub OTA)** 🔄
   - 컴퓨터를 연결할 필요 없이, 톡방에서 `/update` 라고 입력하는 것만으로 개인 GitHub Private 저장소에 업로드된 최신 펌웨어 패키지(.bin)를 읽어와 퍼센트 단위로 다운로드하고 자동으로 기기를 덮어씌웁니다.

## 🛠 하드웨어 핀 맵 (Pinout)

*이 프로젝트는 `ESP32-C3` 마이크로컨트롤러 환경에서 구성되었습니다.*

| 디바이스 (부품) | ESP32-C3 핀 연동 |
| :--- | :--- |
| **RF 433MHz 수신기** (Data 핀) | `GPIO 2` |
| **OLED (I2C 통신)** SDA | `GPIO 18` |
| **OLED (I2C 통신)** SCL | `GPIO 19` |
| **물리 페어링/설정 버튼** | `GPIO 9` |
| **상태 알림 LED** | `GPIO 15` |

## 📖 사용 및 조작 가이드

### 1단계: 초기 인증 및 켜기
- 전원이 들어가면 시스템이 켜지며 화면에 `System Booting...`이 나타납니다.
- 내부 EEPROM 메모리에서 라이선스 키를 대조해 정품임이 확인되면 `WIFI Setup` 단계로 넘어옵니다.

### 2단계: 와이파이 및 텔레그램 연동하기 (최초 1회)
1. 기기에서 빨간색 LED가 빠르게 점멸하며 `SOS_Gateway_Setup` 이라는 와이파이를 뿜어냅니다.
2. 스마트폰이나 노트북으로 이 와이파이에 연결하면 브라우저에 **설정 창(Captive Portal)** 이 뜹니다.
3. 집 안의 2.4GHz 공유기 이름(SSID)을 선택하고 비밀번호를 입력합니다.
4. 아래에 **[Telegram Bot Token]** 과 **[Telegram Chat ID]** 를 적고 `Save`를 누르시면 3초 후 기기가 재부팅되며 인터넷과 톡방에 연결됩니다!

### 3단계: SOS 무선 버튼 등록하기 (페어링)
1. 게이트웨이 기기 본체에 달린 물리 버튼(`GPIO 9`)을 **가볍게 한 번** 딸깍 누릅니다.
2. 화면에 `Pairing Mode` 가 뜨고 20초간 대기 상태가 됩니다.
3. 이 20초 안에 가족들이 누를 동그란 SOS RF 버튼들을 순서대로 차례차례 한 번씩 꾹꾹 눌러줍니다.
   - 누를 때마다 화면 우측 하단 뱃지 숫자가 1, 2, 3... 올라가며 저장됩니다.
4. 더 이상 누를 게 없다면 20초 후 자동으로 대기 모드로 돌아옵니다. (혹시 잘못 눌러서 아무것도 등록하지 않고 20초가 지나면 롤백 기능 덕분에 기존 세팅이 그대로 유지되어 안전합니다.)

### 실전 테스트!
- 등록이 완료된 페어링 버튼을 누르면 OLED 화면에 예쁜 종이비행기가 반짝거리며 텔레그램에 **"🚨 가족의 1번 SOS 버튼이 방금 눌렸습니다!"** 라고 즉시 알림이 발송됩니다.

## 🛠 텔레그램 지원 명령어 목록

단톡방에 초대한 게이트웨이 봇에게 아래 명령어를 치면 기기가 반응합니다.
- `/status` : 기기가 전원이 잘 들어와있는지, 몇 개의 버튼이 등록되어 있고 현재 버전은 몇인지 응답합니다.
- `/update` : 관리자의 비공개 GitHub 서버를 찔러보고, 최신 업데이트 펌웨어가 있다면 자동으로 원격 다운로드하고 시스템을 새로고침합니다. (유령 메시지 방지 알고리즘이 적용되어 안전합니다!)

## 👨‍💻 설계 노트 & 디버깅 (시리얼 모니터 로깅)
- 이 펌웨어는 유지운영(Maintenance)을 위해 모든 과정을 10% 단위의 다운로드 수치, 버그 오류, HTTP 404 상태코드 등의 세세한 정보를 시리얼 모니터(115200bps) 출력으로 상세하게 뿜어냅니다. 운영 도중 문제가 발생할 경우 USB 케이블을 꽂고 시리얼 포트를 모니터링하면 그 즉시 원인을 찾을 수 있도록 최적화되어 있습니다.
